Class: Currency

models/Currency~Currency(currencyObject)

Every expense document in database has reference to particular currency document. Multiple expense documents can have reference to the same currency document.

Currency document property rate matches to equivalent object in rates property of Rate document.

Constructor

new Currency(currencyObject)

currencyObject has 3 properties and all are mandatory:

base - base currency code

date - date of rate

rate - object where key is currency code and value is actual rate

Mongoose new mongoose.Schema({...}, options) creates additional properties: _id, __v, createdAt and updatedAt, first two by default, second two when passing {timesatmps: true} as second argument.

It's mongoose model and the instance is called document.

Models are fancy constructors compiled from Schema definitions. An instance of a model is called a document. Models are responsible for creating and reading documents from the underlying MongoDB database.

Validation is based on currencySchemaObject.

Example

Example usage of Currency model

  base: 'USA',
  date: new Date('2019-11-07'),
  rate['EUR']: 0.89
